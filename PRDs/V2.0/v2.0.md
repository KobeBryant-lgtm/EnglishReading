\# ReadEng V2.0 产品需求文档（PRD）



\---



\## 1. 文档信息



| 项目 | 内容 |

|------|------|

| 产品名称 | ReadEng — 每日英语外刊精读平台 |

| 文档版本 | V2.0 |

| 当前网址 | englishreading.cc.cd |

| 文档日期 | 2026年4月 |

| 数据库 | PostgreSQL |

| 架构 | 前后端分离（RESTful API） |



\---



\## 2. 版本现状与迭代目标



\### 2.1 V1.0 现状



| 功能模块 | 状态 | 问题 |

|---------|------|------|

| 文章首页 | ✅ 每日推荐 + 文章列表（来源/难度筛选） | — |

| 文章阅读 | ✅ 全文阅读、字号调节、全文翻译、点击查词/句翻译 | — |

| 生词本 | ✅ 阅读时点击加入、学习状态筛选 | 数据存 localStorage，换设备丢失 |

| 外刊来源 | ✅ 来源列表展示 | — |

| 文章更新 | ⚠️ 用户手动点击"更新文章"触发爬取 | 不可控、重复触发、体验差 |

| 用户系统 | ❌ 无 | 无法区分用户、无法持久化个人数据 |

| 后台管理 | ❌ 无 | 无法管理文章/用户/爬虫 |



\### 2.2 V2.0 迭代目标



| 序号 | 目标 | 概述 |

|------|------|------|

| 1 | 个性化功能 \& 自动更新 | 文章收藏、阅读历史、生词本云端同步；取消用户端"更新文章"按钮，服务端每日北京时间零点自动抓取 |

| 2 | 用户系统 | 用户名+密码注册（含简单算术验证码防机器人）；登录/注册页面；游客可阅读，登录用户解锁个性化功能 |

| 3 | 后台管理系统 | 前后端分离独立 SPA；管理员登录后自动跳转后台；管理文章、用户、爬虫、数据统计 |



\---



\## 3. 用户角色定义



| 角色 | 说明 | 权限 |

|------|------|------|

| 游客 (Guest) | 未登录用户 | 浏览文章列表、阅读文章全文、查看来源页、使用本地生词本 |

| 注册用户 (User) | 已注册并登录 | 游客全部权限 + 收藏文章、云端生词本、阅读历史、个人中心 |

| 管理员 (Admin) | 后台管理人员 | 登录后自动跳转后台管理系统，拥有全部管理权限 |

\## 4. 功能需求详述



\### 4.1 模块一：个性化功能 \& 文章自动更新



\#### 4.1.1 文章自动更新



| 项目 | 说明 |

|------|------|

| 触发方式 | 服务端定时任务 (Cron Job)，每日北京时间 00:00 自动执行 |

| 爬取逻辑 | 按已配置的外刊来源列表依次爬取当日最新文章 |

| 去重规则 | 基于文章原文 URL 去重 |

| 失败处理 | 记录错误日志，后台可查看；支持后台手动触发重新爬取 |

| 前端变化 | \*\*移除\*\*首页"更新文章"按钮；顶部显示"今日已自动更新 X 篇"提示 |



\#### 4.1.2 文章收藏



| 项目 | 说明 |

|------|------|

| 入口 | ① 文章列表卡片收藏图标 ② 文章阅读页收藏按钮 ③ 阅读页右侧悬浮按钮 |

| 权限 | 仅登录用户；游客点击时弹出提示引导登录 |

| 取消收藏 | 再次点击取消 |

| 收藏列表 | 个人中心 → "我的收藏"，支持按时间排序、来源筛选 |

| 存储 | PostgreSQL，关联 user\_id + article\_id + created\_at |



\#### 4.1.3 阅读历史



| 项目 | 说明 |

|------|------|

| 记录规则 | 登录用户打开文章阅读页自动记录（去重，同篇更新 last\_read\_at） |

| 展示 | 个人中心 → "阅读历史"，按时间倒序 |

| 清除 | 单条删除、一键清空 |



\#### 4.1.4 生词本云端同步



| 项目 | 说明 |

|------|------|

| 现状 | localStorage 本地存储 |

| 迭代 | 登录用户存 PostgreSQL，云端同步；游客仍用本地存储 |

| 首次登录迁移 | 检测本地生词数据，提示合并到云端 |

\### 4.2 模块二：用户系统



\#### 4.2.1 注册



| 项目 | 说明 |

|------|------|

| 方式 | 用户名 + 密码 + 简单算术验证码 |

| 用户名 | 4–20位，字母/数字/下划线，全局唯一 |

| 密码 | 至少6位 |

| 验证码 | 简单算术验证码（如 "3 + 5 = ?"），防止机器人批量注册 |

| 注册成功 | 自动登录并跳转首页 |



\#### 4.2.2 登录



| 项目 | 说明 |

|------|------|

| 方式 | 用户名 + 密码 |

| Token | JWT，access\_token 有效期 24h，refresh\_token 有效期 7d |

| 记住登录 | 勾选"记住我"使用 refresh\_token 自动续期 |

| 管理员登录 | 如果 role=admin，登录后自动跳转后台管理系统 |

| 普通用户登录 | 跳转首页 |



\#### 4.2.3 权限对照



| 功能 | 游客 | 登录用户 |

|------|------|---------|

| 浏览文章列表 | ✅ | ✅ |

| 阅读文章全文 | ✅ | ✅ |

| 查看来源页 | ✅ | ✅ |

| 本地生词本 | ✅ | — |

| 云端生词本 | ❌ → 引导登录 | ✅ |

| 收藏文章 | ❌ → 引导登录 | ✅ |

| 阅读历史 | ❌ | ✅ |

| 个人中心 | ❌ | ✅ |



\#### 4.2.4 个人中心



| 子页面 | 内容 |

|--------|------|

| 个人信息 | 头像、用户名（不可改）、昵称（可改）、注册时间 |

| 我的收藏 | 收藏文章列表，来源筛选 |

| 阅读历史 | 已读文章，支持删除/清空 |

| 生词本 | 同导航栏入口 |

| 账号安全 | 修改密码、退出登录 |



\#### 4.2.5 导航栏状态



| 状态 | 导航栏右侧 |

|------|-----------|

| 未登录 | `登录` + `注册` 按钮 + 🌙 |

| 已登录 | 🌙 + 用户头像（hover下拉：个人中心、我的收藏、阅读历史、退出登录） |



\#### 4.2.6 页面跳转规则



| 动作 | 跳转目标 |

|------|---------|

| 点击"登录"按钮 | 跳转登录页 |

| 点击"注册"按钮/登录页"立即注册" | 跳转注册页 |

| 登录成功（普通用户） | 跳转首页 |

| 登录成功（管理员） | 跳转后台管理系统 |

| 注册成功 | 自动登录，跳转首页 |

| 点击"退出登录" | 清除 token，跳转首页 |

| 游客点击收藏/云端生词本 | 弹出提示，引导跳转登录页 |

\## 5. 数据库设计（PostgreSQL）



\### 5.1 users — 用户表



```sql

CREATE TABLE users (

&#x20;   id            SERIAL PRIMARY KEY,

&#x20;   username      VARCHAR(20) UNIQUE NOT NULL,

&#x20;   password\_hash VARCHAR(255) NOT NULL,

&#x20;   nickname      VARCHAR(50),

&#x20;   avatar\_url    VARCHAR(500),

&#x20;   role          VARCHAR(10) DEFAULT 'user' CHECK (role IN ('user', 'admin')),

&#x20;   status        VARCHAR(10) DEFAULT 'active' CHECK (status IN ('active', 'disabled')),

&#x20;   created\_at    TIMESTAMP DEFAULT NOW(),

&#x20;   updated\_at    TIMESTAMP DEFAULT NOW(),

&#x20;   last\_login\_at TIMESTAMP

);

CREATE INDEX idx\_users\_username ON users(username);

```



\### 5.2 articles — 文章表



```sql

CREATE TABLE articles (

&#x20;   id                 SERIAL PRIMARY KEY,

&#x20;   title              VARCHAR(500) NOT NULL,

&#x20;   summary            TEXT,

&#x20;   content            TEXT,

&#x20;   source\_name        VARCHAR(100),

&#x20;   source\_url         VARCHAR(1000) UNIQUE,

&#x20;   difficulty         VARCHAR(20),

&#x20;   word\_count         INTEGER DEFAULT 0,

&#x20;   cover\_image\_url    VARCHAR(500),

&#x20;   is\_daily\_recommend BOOLEAN DEFAULT FALSE,

&#x20;   view\_count         INTEGER DEFAULT 0,

&#x20;   is\_deleted         BOOLEAN DEFAULT FALSE,

&#x20;   published\_at       TIMESTAMP,

&#x20;   created\_at         TIMESTAMP DEFAULT NOW(),

&#x20;   updated\_at         TIMESTAMP DEFAULT NOW()

);

CREATE INDEX idx\_articles\_source ON articles(source\_name);

CREATE INDEX idx\_articles\_difficulty ON articles(difficulty);

CREATE INDEX idx\_articles\_created ON articles(created\_at DESC);

```



\### 5.3 user\_favorites — 收藏表



```sql

CREATE TABLE user\_favorites (

&#x20;   id         SERIAL PRIMARY KEY,

&#x20;   user\_id    INTEGER REFERENCES users(id) ON DELETE CASCADE,

&#x20;   article\_id INTEGER REFERENCES articles(id) ON DELETE CASCADE,

&#x20;   created\_at TIMESTAMP DEFAULT NOW(),

&#x20;   UNIQUE(user\_id, article\_id)

);

CREATE INDEX idx\_fav\_user ON user\_favorites(user\_id);

```



\### 5.4 reading\_history — 阅读历史表



```sql

CREATE TABLE reading\_history (

&#x20;   id           SERIAL PRIMARY KEY,

&#x20;   user\_id      INTEGER REFERENCES users(id) ON DELETE CASCADE,

&#x20;   article\_id   INTEGER REFERENCES articles(id) ON DELETE CASCADE,

&#x20;   last\_read\_at TIMESTAMP DEFAULT NOW(),

&#x20;   UNIQUE(user\_id, article\_id)

);

CREATE INDEX idx\_history\_user ON reading\_history(user\_id);

```



\### 5.5 vocabulary — 生词表



```sql

CREATE TABLE vocabulary (

&#x20;   id                SERIAL PRIMARY KEY,

&#x20;   user\_id           INTEGER REFERENCES users(id) ON DELETE CASCADE,

&#x20;   word              VARCHAR(100) NOT NULL,

&#x20;   translation       VARCHAR(500),

&#x20;   context\_sentence  TEXT,

&#x20;   source\_article\_id INTEGER REFERENCES articles(id) ON DELETE SET NULL,

&#x20;   status            VARCHAR(10) DEFAULT 'learning' CHECK (status IN ('learning', 'mastered')),

&#x20;   created\_at        TIMESTAMP DEFAULT NOW(),

&#x20;   updated\_at        TIMESTAMP DEFAULT NOW()

);

CREATE INDEX idx\_vocab\_user ON vocabulary(user\_id);

```



\### 5.6 crawl\_tasks — 爬虫任务表



```sql

CREATE TABLE crawl\_tasks (

&#x20;   id               SERIAL PRIMARY KEY,

&#x20;   source\_name      VARCHAR(100),

&#x20;   trigger\_type     VARCHAR(10) CHECK (trigger\_type IN ('auto', 'manual')),

&#x20;   status           VARCHAR(10) CHECK (status IN ('running', 'success', 'failed')),

&#x20;   articles\_fetched INTEGER DEFAULT 0,

&#x20;   error\_log        TEXT,

&#x20;   started\_at       TIMESTAMP,

&#x20;   finished\_at      TIMESTAMP

);

```



\### 5.7 sources — 来源表



```sql

CREATE TABLE sources (

&#x20;   id          SERIAL PRIMARY KEY,

&#x20;   name        VARCHAR(100) NOT NULL,

&#x20;   category    VARCHAR(50),

&#x20;   feed\_url    VARCHAR(1000),

&#x20;   is\_active   BOOLEAN DEFAULT TRUE,

&#x20;   description TEXT,

&#x20;   created\_at  TIMESTAMP DEFAULT NOW(),

&#x20;   updated\_at  TIMESTAMP DEFAULT NOW()

);

```

\## 6. API 设计



\### 6.1 用户端 API



```

\# === 认证 ===

POST   /api/auth/register           # 请求体: { username, password, captcha\_id, captcha\_answer }

POST   /api/auth/login               # 请求体: { username, password }

POST   /api/auth/refresh             # 请求体: { refresh\_token }

GET    /api/auth/captcha             # 响应: { captcha\_id, question: "3 + 5 = ?" }



\# === 用户信息 ===

GET    /api/user/profile             # 获取当前用户信息

PUT    /api/user/profile             # 更新: { nickname, avatar\_url }

PUT    /api/user/password            # 修改密码: { old\_password, new\_password }



\# === 文章 ===

GET    /api/articles                 # 查询参数: source, difficulty, page, limit

GET    /api/articles/:id             # 文章详情

GET    /api/articles/daily-recommend # 每日推荐



\# === 收藏 ===

GET    /api/favorites                # 查询参数: page, source

POST   /api/favorites/:articleId     # 收藏

DELETE /api/favorites/:articleId     # 取消收藏

GET    /api/favorites/check/:articleId # 检查是否已收藏



\# === 阅读历史 ===

GET    /api/history                  # 查询参数: page

POST   /api/history/:articleId       # 记录阅读

DELETE /api/history/:id              # 删除单条

DELETE /api/history                  # 清空全部



\# === 生词本 ===

GET    /api/vocabulary               # 查询参数: status, search, page

POST   /api/vocabulary               # 添加生词

PUT    /api/vocabulary/:id           # 更新（改状态等）

DELETE /api/vocabulary/:id           # 删除

POST   /api/vocabulary/merge-local   # 合并本地数据到云端



\# === 来源 ===

GET    /api/sources                  # 来源列表

```

\## 8. 页面清单与跳转关系



\### 8.1 用户端页面



| 页面 | 路由 | 说明 |

|------|------|------|

| 首页 | `/` | 改造：移除更新按钮、增加收藏、导航栏登录状态 |

| 来源 | `/sources` | 不变 |

| 登录 | `/login` | 新增：用户名+密码 |

| 注册 | `/register` | 新增：用户名+密码+算术验证码 |

| 个人中心 | `/profile` | 新增：收藏/历史/信息/安全 |



\### 8.2 管理后台页面



| 页面 | 路由 | 说明 |

|------|------|------|

| 文章管理 | `/admin/articles` | 文章 CRUD |

\### 8.3 页面跳转流程



```

首页 → 点击"登录" → 登录页

首页 → 点击"注册" → 注册页

登录页 → 点击"立即注册" → 注册页

注册页 → 点击"立即登录" → 登录页

登录成功(role=user) → 首页

登录成功(role=admin) → 后台管理系统

注册成功 → 自动登录 → 首页

退出登录 → 首页（游客状态）

游客点击收藏 → 弹窗"请先登录" → 可跳转登录页

后台左侧栏菜单 → 切换管理面板

后台 → 点击"返回前台" → 首页

```

\## 10. 附录：页面 Demo



见独立 HTML 文件 `readeng-v2-demo.html`，包含全部页面的可交互原型：



1\. \*\*首页\*\*（游客/登录态切换、收藏交互、自动更新提示、无更新按钮）

2\. \*\*登录页\*\*（用户名+密码、记住我、跳转注册）

3\. \*\*注册页\*\*（用户名+密码+算术验证码、跳转登录）

4\. \*\*个人中心\*\*（侧边栏导航：我的收藏/阅读历史/个人信息/账号安全）

5\. \*\*文章阅读页\*\*（收藏按钮、悬浮操作栏、返回列表）

6\. \*\*后台管理系统\*\*（仪表盘/文章管理/用户管理/爬虫任务/来源管理，侧边栏切换）

| 操作 | 效果 |

|------|------|
